# ArchiGent: Next-Generation Agentic BIM Workflow

## 1. 프로젝트 개요 (Overview)
**ArchiGent**는 복잡하고 거대한 BIM(Building Information Modeling) 데이터 표준인 **IFC(Industry Foundation Classes)** 파일을 자연어 지시만으로 분석, 검색, 수정할 수 있도록 설계된 **AI 에이전틱 워크플로우(Agentic Workflow)** 시스템입니다. 본 프로젝트는 건축 도메인의 전문 지식과 최신 LLM(Large Language Model)의 추론 능력을 결합하여, 누구나 코딩 없이도 건축 모델을 자유롭게 제어할 수 있는 지능형 인터페이스를 지향합니다.

---

## 2. 타겟 및 니즈 (Target & Needs)

### 🎯 핵심 타겟
1. **AEC(건축, 건설, 엔지니어링) 실무자**: 가시화 도구의 복잡한 메뉴 조작 대신 "벽체 두께를 한꺼번에 바꿔줘"와 같은 직관적인 명령으로 업무 효율을 높이려는 설계자.
2. **BIM 품질 관리자(QA/QC)**: 수많은 IFC 파일 내에서 특정 조건(예: 소방 법규 미준수 구획 등)을 검색하고 자동으로 보정하려는 관리자.
3. **건축 IT 솔루션 개발자**: LLM과 BIM 커널을 결합한 새로운 비즈니스 로직을 구축하려는 엔지니어.

### ⚠️ 타겟의 문제 상황 (Pain Points)
- **IFC의 극심한 복잡성**: 수만 개의 엔티티와 깊은 관계망으로 얽힌 IFC 데이터 구조는 일반적인 API만으로는 접근 및 수정이 매우 어렵습니다.
- **높은 진입 장벽**: `IfcOpenShell`과 같은 전문 라이브러리를 숙달하는 데 많은 시간과 기술적 배경이 요구됩니다.
- **수정 결과의 불확실성**: 단순한 코드 생성만으로는 실제 BIM 데이터가 의도대로 바뀌었는지 검증하기 어렵고, 오류 발생 시 모델 전체가 파괴될 위험이 있습니다.

---

## 3. 문제 해결 방안 (Solution)
ArchiGent는 **"분석 - 계획 - 수행 - 검증"**의 4단계 순환 루프를 통해 이 문제를 해결합니다.
- **그래프 기반 컨텍스트 추출**: Neo4j를 활용해 IFC의 복잡한 횡적/종적 관계를 그래프로 구조화하여 LLM이 데이터를 정확히 인지하게 합니다.
- **자가 치유형 코딩(Self-Correction)**: 생성된 코드가 오류를 일으키면 스스로 디버깅하여 성공할 때까지 반복 수행합니다.
- **확정적 검증(Deterministic Verification)**: 수정 결과가 원래 계획과 일치하는지 규칙 기반(Rule-based)으로 엄격하게 대조하여 신뢰성을 확보합니다.

---

## 4. 상세 파이프라인 (Pipeline Detail)

ArchiGent의 워크플로우는 **LangGraph**를 통해 상태(State)를 관리하며 유기적으로 작동합니다.

1. **DB 초기화 & 로딩 (Initialize)**:
   - 선택된 IFC 파일을 파싱하여 **Neo4j 그래프 DB**에 노드와 관계로 변환하여 적재합니다.
2. **분석 에이전트 (Analyzer)**:
   - 사용자의 자연어를 해석하여 최적의 **Cypher 쿼리**를 생성, DB에서 관련 객체 정보를 추출하여 LLM용 컨텍스트(JSON)를 구축합니다.
3. **기획 에이전트 (Planner)**:
   - 분석 결과와 요청 의도를 바탕으로 **조회(QUERY)** 또는 **수정(MODIFY)** 모드를 판별합니다.
   - 수정 시에는 3가지 건축적 시안을 제안하며, 사용자의 선택(HITL)을 받아 최종 작업 명세(Task Spec)를 확정합니다.
4. **코딩 에이전트 (Coder)**:
   - 수립된 계획과 `IfcOpenShell` RAG 지식 기반을 참고하여 실제 Python 코드를 작성합니다.
5. **실행 및 자가 수정 (Sandbox)**:
   - 격리된 환경(Sandbox)에서 코드를 실행합니다. 런타임 에러 발생 시 **Reviewer Agent**와 협력하여 코드를 수정합니다.
6. **검험 및 검증 (Review & Verify)**:
   - **Reviewer**: 코드의 안전성과 계획 준수 여부를 논리적으로 검토합니다.
   - **Verifier**: 수정된 결과 파일과 원본 파일을 실질적으로 비교하여 물리적 변경 사항을 최종 확정합니다.
7. **환류 및 롤백 (Feedback & Rollback)**:
   - 검증 실패 시 작업 이전 상태로 **Rollback**하고, 실패 정보를 담아 다시 기획 단계로 돌아가 성공할 때까지 반복합니다.

## 5. 기술 스택 (Tech Stack)

| 구분 | 기술 요소 | 비고 |
| :--- | :--- | :--- |
| **LLM** | **Google Gemini 2.5 Flash** | 고속 추론 및 대규모 컨텍스트 처리 |
| **Logic** | **LangGraph** | 상태 기반 멀티 에이전트 오케스트레이션 |
| **Database** | **Neo4j** | IFC 객체 관계망 구조화 및 고속 검색 |
| **BIM Kernel** | **IfcOpenShell** | IFC 2x3/4/4x3 표준 처리 및 수정 |
| **Execution** | **Python Sandbox** | 안전한 외부 코드 실행 환경 |
| **Environment** | **uv** (Python Package Manager) | 초고속 가상환경 및 의존성 관리 |

## 6. 기대 효과 및 최종 목표

### 🚀 기대 효과
- **생산성 극대화**: 수동으로는 수 시간 걸리는 복합 수정을 단 몇 분 내에 완료할 수 있습니다.
- **데이터 일관성**: 규칙 기반 검증을 통해 대규모 모델 수정 시에도 데이터의 무결성을 100% 유지합니다.
- **의사결정 지원**: 자연어 대화만으로 모델의 속성 정보를 입체적으로 분석하여 데이터 기반의 의사결정을 돕습니다.

### 🏁 최종 목표 (The Vision)
ArchiGent의 최종 목표는 단순한 툴을 넘어 **"건축 지능의 자동화"**를 실현하는 것입니다. 향후에는 시공성 검토, 에너지 시뮬레이션 최적화, 건축 법규 자가 진단 등 고도의 공학적 판단이 필요한 영역까지 확장하여, 인간 건축가와 AI 에이전트가 공생하는 차세대 BIM 생태계를 구축하고자 합니다.
